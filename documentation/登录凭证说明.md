# 登录凭证说明 (Login Credentials Guide)

## 系统架构说明

本项目包含三个独立的应用系统，每个系统有不同的登录凭证：

### 1. 前端应用 (Frontend) - 端口 3000
**用途**: 模拟Amazon Seller Central的主应用，用于卖家查看销售数据、管理产品等

**登录凭证** (从 `backend/data/users.json` 读取):

| 邮箱 | 密码 | OTP验证码 | 角色 | 说明 |
|------|------|---------|------|------|
| admin@technest.com | A123456 | 123456 | Admin | 系统管理员 |
| admin@example.com | password123 | 123456 | Admin | 演示管理员 |
| manager@technest-us.com | B654321 | 654321 | Manager | 美国店铺经理 |
| manager@technest-jp.com | C789123 | 789123 | Manager | 日本店铺经理 |
| staff1@technest-us.com | D456789 | 456789 | User | 美国客服专员 |
| staff2@technest-jp.com | E987654 | 987654 | User | 日本客服专员 |

**登录流程**:
1. 访问 http://localhost:3000
2. 输入邮箱地址 (例如: admin@technest.com)
3. 点击"继续"
4. 输入密码 (例如: A123456)
5. 点击"登录"
6. 输入OTP验证码 (例如: 123456)
7. 点击"登录"完成

**用途**: 管理员后台，用于配置系统数据、管理店铺、用户等

**登录凭证** (固定凭证):

| 用户名 | 密码 |
|--------|------|
| admin | admin123 |

**登录流程**:
1. 访问 http://localhost:3002
2. 输入用户名: `admin`

### 3. 后端API服务 (Backend API) - 端口 3001
**用途**: RESTful API服务，为前端和管理后台提供数据接口

**主要端点**:

- 用户: `/api/users`

## 常见问题

### Q: 为什么前端登录失败 (401 Unauthorized)?
**A**: 确保你使用的是前端用户凭证，而不是管理后台凭证。
- ❌ 错误: 使用 `admin` / `admin123` (这是管理后台凭证)
- ✅ 正确: 使用 `admin@technest.com` / `A123456` (这是前端用户凭证)

### Q: 管理后台登录失败?
**A**: 确保使用正确的管理后台凭证:
- 用户名: `admin`
- 密码: `admin123`

### Q: 如何添加新用户?
**A**: 
1. 登录管理后台 (http://localhost:3002)
2. 进入"用户管理"页面
3. 点击"添加用户"
4. 填写用户信息并保存
5. 新用户可以使用设置的邮箱和密码登录前端

### Q: OTP验证码是什么?
**A**: OTP (One-Time Password) 是一次性密码，用于增强安全性。在本演示系统中，OTP验证码已预设在用户数据中。实际生产环境中应使用TOTP或其他安全的OTP生成方式。

### Q: 如何修改用户密码?
**A**:
1. 登录管理后台
2. 进入"用户管理"页面
3. 找到要修改的用户
4. 点击编辑，修改密码
5. 保存更改

## 安全提示

⚠️ **注意**: 这是演示项目，不应用于生产环境。生产环境应该:
- 使用加密的密码存储 (bcrypt, argon2等)
- 实现真正的JWT token认证
- 使用TOTP或其他安全的OTP方案
- 启用HTTPS
- 实现速率限制和防暴力破解
- 添加审计日志
