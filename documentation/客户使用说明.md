# Amazon Seller Central - 客户版使用说明

## 🚀 快速开始

### 第一步：安装
1. 右键以管理员身份运行 `install.bat`
2. 等待安装完成（可能需要几分钟）
3. 安装完成后会在桌面创建快捷方式

### 第二步：启动
选择以下任一方式启动：
- **推荐**：双击桌面快捷方式 "Amazon Seller Central"
- 双击 `smart-launcher.bat` (智能启动器)
- 双击 `launcher.bat` (基础启动器)

## 🌐 启动模式

### 本地模式 (推荐)
- **适用场景**：大多数用户
- **特点**：启动最快，最稳定
- **访问地址**：
  - 前端应用：http://localhost:3000
  - 管理后台：http://localhost:3002

### 域名模式 (高级)
- **适用场景**：需要自定义域名的高级用户
- **特点**：支持自定义域名，模拟真实Amazon环境
- **启动方式**：
  - 使用智能启动器选择域名模式
  - 或运行 `node 一键配置域名.js`

## 🛠️ 故障排除

### 常见问题

#### 1. 启动失败
**症状**：点击启动器没有反应或报错
**解决方案**：
1. 运行 `troubleshoot.bat` 进行诊断
2. 检查是否以管理员身份运行
3. 确认Node.js已正确安装

#### 2. 端口被占用
**症状**：提示端口3000、3001、3002被占用
**解决方案**：
1. 运行故障排除工具选择"强制启动服务"
2. 或手动结束占用端口的进程

#### 3. 网页无法访问
**症状**：浏览器显示"无法访问此网站"
**解决方案**：
1. 等待服务完全启动（首次启动需要更长时间）
2. 检查防火墙设置
3. 尝试刷新浏览器

#### 4. 依赖安装失败
**症状**：安装过程中出现npm错误
**解决方案**：
1. 以管理员身份重新运行安装
2. 临时关闭杀毒软件
3. 检查网络连接

### 故障排除工具
运行 `troubleshoot.bat` 可以：
- 全面系统诊断
- 清理并重新安装依赖
- 修复Node.js环境变量
- 检查安全软件冲突
- 测试网络连接
- 强制启动服务
- 生成诊断报告

## 🔧 高级配置

### 域名配置
如需使用自定义域名：
1. 运行 `node 一键配置域名.js`
2. 选择域名模式

1. 访问管理后台
2. 进入对应的数据配置页面

## 💡 使用提示

### 性能优化
- 首次启动需要较长时间，请耐心等待
- 建议关闭不必要的后台程序
- 确保有足够的磁盘空间（至少500MB）

### 安全建议
- 仅在受信任的网络环境中使用
- 定期备份重要数据
- 不要在生产环境中使用

### 最佳实践
- 使用Chrome或Edge浏览器获得最佳体验
- 定期清理浏览器缓存
- 遇到问题先尝试重启服务

## 📞 技术支持

### 自助诊断
1. 运行 `troubleshoot.bat`
2. 查看生成的诊断报告
3. 根据提示解决问题

### 日志查看
- 后端日志：查看后端服务窗口输出
- 前端错误：按F12打开浏览器开发者工具

### 常用命令
```bash
# 查看Node.js版本
node --version

# 查看npm版本
npm --version

# 清理npm缓存
npm cache clean --force

# 重新安装依赖
npm install
```

## 📋 系统要求

### 最低要求
- **操作系统**：Windows 10 或更高版本
- **内存**：4GB RAM
- **磁盘空间**：1GB 可用空间

- **内存**：8GB RAM 或更高

- **浏览器**：Chrome 90+ 或 Edge 90+

- **Node.js**：16.0 或更高版本（安装程序会自动检测和提示）
